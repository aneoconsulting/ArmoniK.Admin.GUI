Comprendre et utiliser l'application

Cette page vous permet de comprendre l'ensemble des fonctionnalités présentes au sein de l'interface afin de l'utiliser au mieux.

Afficher des données

Différentes données, comme les sessions, les tâches, sont affichées dans différentes tables présentes au sein de l'application.

Ces tables permettent de visualiser, trier et filtrer les données. Une colonne est triable dès lors que l'icône pour filtrer est visible. Pour trier une colonne, il faut cliquer sur le nom de cette dernière. Pour filtrer une colonne, il faut cliquer sur l'icône .

Naviguer entre les pages

La navigation au sein de l'application se fait de différentes manières.

  • Via le tableau de bord. Ce dernier liste l'ensemble des applications au sein d'ArmoniK. Il est ensuite possible de naviguer au sein des sessions d'une application puis au sein des tâches d'une session.
  • Via les onglets. Ces derniers permettent de passer rapidement d'une application à une autre dès que vous y avez déjà été.

Trouver les erreurs

Il peut arriver qu'une erreur survienne lors de l'utilisation de l'application. Cela peut se manifester par l'apparition d'un bandeau rouge ou par l'affichage d'une page d'erreur.

Lorsque cela se produit, il est possible d'en savoir plus sur l'erreur via la console et l'onglet réseau de l'inspecteur du navigateur.
Lors d'une remontée de bogues, il est important de transmettre cette erreur ainsi que le numéro de version de l'application.

Changer la langue

L'application est disponible en 2 langues : français et anglais. Pour la changer, il suffit simplement de cliquer sur le bouton correspondant dans l'en-tête de la page, en haut à droite.

La langue actuellement utilisée n'est plus disponible à la sélection et son affichage est grisé. Aussi, votre sélection est gardée en mémoire pour vos prochaines navigations au sein de l'application.

Voir la version

La version de l'application est affichée dans la barre de navigation en bas à gauche. Elle permet de donner des informations sur la version de l'application utilisée.

Il est important de transmettre cette version à l'équipe de développement lors d'une remonté de bogues afin de faciliter la correction de ces derniers.

Understanding and using the application

This page allows you to understand all the functionalities present in the interface in order to use it at best.

Display data

Different data, such as sessions, tasks, are displayed in different tables within the application.

These tables allow you to view, sort and filter data. A column is sortable as soon as the icon to filter is visible. To sort a column, click on the name of the column. To filter a column, click on the icon .

Navigate between pages

The navigation within the application is done in different ways.

  • Via the dashboard. The latter lists all the applications within ArmoniK. It is then possible to navigate within the sessions of an application and then within the tasks of a session.
  • Via the tabs. These tabs allow you to quickly switch from one application to another once you've been there.

Find the errors

It may happen that an error occurs while using the application. This can be shown by the appearance of a red banner or by the display of an error page.

When this happens, it is possible to find out more about the error via the console and the network tab of the browser inspector.
When reporting bugs, it is important to forward this error as well as the version number of the application.

Change the language

The application is available in 2 languages: French and English. To change it, you just have to click on the corresponding button in the the header of the page, on the top right.

The language currently in use is no longer available for selection and its selection and displayed is grayed out color. Also, your selection is kept in memory for your next future navigations within the application.

See the version

The version of the application is displayed in the bottom left navigation bar. It gives information about the version of the application of the application used.

It is important to to forward this version to the development team when a bug is reported in order to facilitate the correction of these the latter.
